# Contributing to DryGram

We welcome contributions to the DryGram project!

## How to Contribute
1. Fork the repository.
2. Create a new branch for your feature or bugfix.
3. Write clean, type-hinted code following our conventions.
4. Ensure tests are added for any new functionality.
5. Submit a pull request.

## Design Rules
- All new files must contain the correct Santhu author header.
- Avoid using comments inside codebase files except where required.
- Do not introduce placeholder implementations.
